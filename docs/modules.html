gol¶

A simulation of Conway’s Game of Life.

The simulation is visualized using Matplotlib and saved as an animated, infinitely looping GIF.

class gol.Game(rows, columns, survival_rules=[2, 3, 4, 5], birth_rules=[3], initial_state='heart')[source]¶

Manages the state and rules of a Game of Life simulation.

This class holds the grid of cells and provides the mechanism to advance the simulation to the next generation.

update() → None[source]¶

Advances the simulation by one generation according to configurable rules.

This method calculates the number of live neighbors for each cell on the grid using a highly optimized 2D convolution from the SciPy library. It then applies the rules defined in the survival_rules and birth_rules attributes of the class to the entire grid using efficient vectorized NumPy operations.

This approach replaces the slower nested for-loops, making the simulation significantly more performant, especially for larger grids. The use of configurable rules allows the same code to simulate different “universes” or rule sets (e.g., standard Conway’s Life, HighLife, Day & Night) by simply passing different lists during initialization.

The core logic is as follows:

  • A dead cell becomes alive if its neighbor count is in self.birth_rules.

  • A live cell survives if its neighbor count is in self.survival_rules.

  • All other cells die or remain dead.
